What a Drift program looks like

agent InboxTriage {
  model: "gpt-5.4-nano"
  budget: $0.10 per run

  step triage(emails: list<string>) -> list<EmailAnalysis> {
    let analyses = []
    for each email in emails parallel {
      let analysis = classify email as EmailAnalysis
      analyses.add(analysis)
    }
    for each a in analyses {
      if a.priority == "urgent" {
        respond "URGENT — {a.subject}: {a.summary}"
      }
    }
    return analyses
  }
}

That's a full agent: model choice, budget cap, parallel fan-out, structured classification, conditional output, typed return. Ran live against OpenAI: 5 emails, 1.82s, $0.0092, returned 5 dataclasses.

Why a language

Agent code is mostly orchestration: pick a model, call it with structure, branch on the result, escalate if confidence is low, retry on rate limits, track costs. That orchestration reads worse than the prose that describes it.

Drift makes the prose runnable. The transpiler emits async Python that runs on a thin runtime handling provider routing, budget caps, schema validation, retries, memory, and MCP tool calls. You write the spec; the runtime handles the plumbing.

What's in the box

  • Multi-provider routing: prefer "claude-sonnet" fallback "gpt-5.4-nano". OpenAI + Anthropic out of the box, MCP tool support, REST inline.
  • Confidence-gated branching: if scored is confident { … } otherwise { fail "review" }. Cheap model when sure, escalate when not.
  • Strict JSON output: schemas become provider-side JSON Schema constraints. The "model returned almost-JSON" failure mode is gone.
  • Parallel by default: for each x in xs parallel compiles to asyncio.gather.
  • Built-in cost tracking: every run prints a cost report. Hard budget caps.
  • Memory: Dendric (semantic, persistent) or SQLite. remember / recall / deja_vu / forget.
  • Structured error handling: attempt { … } recover from { RateLimited -> retry, BudgetExceeded -> fail "…" }.
  • Developer tools: drift run, drift check, drift fmt, drift run --watch, VS Code syntax highlighting, source-mapped runtime errors.

30 seconds to your first agent

pip install drift-lang
drift new hello
cd hello
drift run hello.drift --input '{"name":"Riley"}'

No API key required — Drift falls back to a mock provider so you see something work immediately. Drop an OPENAI_API_KEY or ANTHROPIC_API_KEY into .env for real LLM calls.

For coding agents

If you're a coding agent (Claude, Cursor, Copilot) writing Drift on a user's behalf: load LLM.md first. It's a single-file complete reference packed for one-shot consumption — every keyword, every type, every runtime behavior, every anti-pattern. Cross-checked against the parser; cold-start agents have written working programs from it on first try.
